# Engine naming — decision + alternatives

**Decision (2026-06-10, working name):** the corpus-agnostic engine package is **`gloss`**
(`src/gloss/`, CLI `gloss`). A *gloss* is an explanatory note attached to a passage of text —
exactly what the engine adds to verbatim passages (`context_line`, `applies_when`, `key_terms`).
The APOSD *instance* stays `corpora/aposd/`; its built artifact stays `aposd.db`.

Working title — may change. Alternatives considered, kept here in case:

- **florilegium** (CLI alias `flor`) — literally "a curated collection of excerpts from texts";
  the most precise name for the artifact we build. Knock: long / obscure to type.
- **concordance** — a searchable index of a text's passages; the most literally accurate name
  for the retrieval index. Knock: a touch long.
- **lectern** — the stand that holds the book and hands you the passage on demand; a
  consultation metaphor. Short, uncommon in software.
- **vade** (from *vade mecum*, "go with me") — the reference you carry everywhere; fits the
  portability thesis (drop the `.db` into any repo). Distinctive but obscure.
- **cite** — plain and practical: the value delivered is citable primary-source passages.
- **gloss** — *chosen.*
